

Deserialize/Unpack for BinaryMarshaller

При обработке получения объекта из бинарного массива тот, кто обрабатывает, должен
только обработать и деаллоцировать объект обратно. НЕЛЬЗЯ чистить массивы и map.
Так как аллокация указателей и умных указателей, возможно производилась с помощью внутреннего аллокатора.
Ответственность за жизненный цикл лежит на BinaryMarshaller.

Для случая упаковки объекта в бинарный массив такого не происходит. Ответственность за жизненный цикл объекта
лежит полностью на разработчике.


BinaryMarshaller - use MemoryPool cause need high speed.
JsonSerializer - not use MemoryPool cause nevermind for speed.
